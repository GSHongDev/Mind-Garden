# 项目命名分析报告

## 一、项目核心特征分析

### 1.1 技术架构特征
- **基础框架**：企业级快速开发平台（前后端分离架构）
- **核心功能**：流媒体管理与视频监控
- **协议支持**：GB/T 28181-2016、ONVIF、RTSP、ISUP等
- **架构模式**：前后端分离（Spring Boot + Vue3）
- **集成组件**：WVP流媒体服务、ZLMediaKit媒体网关

### 1.2 功能定位特征
- **管理功能**：设备管理、用户权限、系统监控
- **媒体功能**：实时视频预览、录像回放、流媒体分发
- **业务场景**：视频监控、安防管理、智能分析

### 1.3 创新点特征
- **深度定制**：企业级框架的二次开发与流媒体集成
- **权限融合**：权限体系与流媒体资源的动态绑定
- **模块化扩展**：Maven多模块架构，支持灵活扩展

## 八、总结

**推荐采用分层命名策略**，在不同场景使用不同名称：

1. **学术论文**：使用"流媒体权限融合管理系统（SM-PFMS）"（突出创新点）
2. **代码仓库**：使用"Streaming-Media-Platform"（突出核心功能）
3. **系统界面**：使用"智能视频分析平台（IVAP）"（与代码一致，用户友好）

这种分层命名策略既保证了学术规范性，又兼顾了技术一致性和用户友好性，同时避免了特定框架的依赖表述，更加通用和独立。
